## 1. 官方发布主文案

### 1.1 预热版

**AgentCore OS v1.0.0 已正式进入稳定版。**

这不是一个只会聊天的 AI 工具，而是一个把模型、文件、连接器、审批、任务流和执行链真正放进工作系统里的本地优先 AI 工作底座。

在 `v1.0.0` 版本中，AgentCore OS 已完成核心产品边界收口：

- 前端主助手命名统一
- LobsterAI 基座执行层完成统一与收口
- 默认执行链路、桌面壳、核心 API 与运行时命名保持一致
- 稳定版发布门禁已经完整通过

当前稳定版重点支持三条最核心的真实业务链路：

- 内容创作
- 销售跟进
- 研究分析

如果你正在寻找的不是一个“聊天页面”，而是一套真正能接入日常工作的本地 AI 工作系统，AgentCore OS 已经进入值得认真体验和推广的阶段。

### 1.2 正式版

**AgentCore OS v1.0.0 正式发布。**

AgentCore OS 是一个本地优先、面向真实工作的 AI 工作底座。它不只是让模型回答问题，而是把模型能力接入到实际工作里，让文件、流程、工具、审批与执行链能在同一个系统中协同。

`v1.0.0` 标志着 AgentCore OS 的第一条稳定产品线正式成立：

- 稳定支持浏览器壳与桌面壳
- 默认采用 BYOK / API Key 驱动模式
- 默认聚焦内容创作、销售跟进、研究分析三条核心工作链
- LobsterAI 基座执行层已完成统一命名与运行时收口
- 默认本地优先，不依赖复杂本地编排即可开始使用

这意味着，个人、团队、工作室和小微企业，已经可以用更低门槛的方式，把 AI 真正放进自己的日常业务流程中。

## 4. 核心卖点

### 4.1 市场版 6 点

1. **本地优先**
   更适合重视可控性、隐私性和长期工作资产沉淀的用户。

2. **不只是聊天**
   AgentCore OS 的目标不是“问答”，而是让 AI 进入真实业务流程。

3. **三条核心链路已稳定**
   当前最成熟的是内容创作、销售跟进、研究分析。

4. **前后端与执行层统一**
   前台产品命名、基座执行层、运行时接口已经统一收口，产品边界更清晰。

5. **默认 BYOK，门槛更低**
   先填入模型供应商、Base URL、模型名和 API Key，就能走通默认体验路径。

6. **桌面壳 + 浏览器壳双入口**
   更贴近真实工作的操作方式，而不是单一网页试玩模式。

### 4.2 更技术一点的卖点

- 基座执行层完成统一命名与边界收口
- LobsterAI 作为 capability substrate / control plane 被明确化
- `runtime-executor` 作为活动执行身份已统一
- IM bridge、scheduler、MCP、skills 与 selected runtime execution paths 已进入同一运行时口径
- 默认 `desktop_light` 稳定线已完成发布门禁复验

不是。

当前更适合宣传为：

- 完整多租户 SaaS

不是。

- 配置模型供应商
- 配置 Base URL

- 配置 API Key

然后就可以开始体验默认主线。
