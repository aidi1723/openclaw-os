# AgentCore OS v1.1.0 对外发布文案

这份文案用于 GitHub Release 正文、社媒发布、官网更新、微信群/社群转发与合作方同步。

这一版的核心口径不要再强调“应用很多”，而要强调：

- 销售和客服两条高频链路更稳了
- 数字员工能力被收敛到少量高价值角色
- 知识资产已经能沉淀、编辑、复用、追溯

不要使用的口径：

- 不要宣传成“全自动无人公司已经成熟”
- 不要宣传成“一个版本覆盖所有行业全部流程”
- 不要承诺“完全免人工审核”

当前最稳的对外定位是：

- 本地优先 AI 工作底座
- 面向真实工作流程，不只是聊天
- 强调销售、客服、知识资产沉淀这三条高频闭环
- 适合个人、团队、工作室、小微企业和内部数字化探索场景

## 1. 官方发布主文案

### 1.1 正式版

**AgentCore OS v1.1.0 正式发布。**

这不是一个靠堆很多 Agent 角色来制造复杂感的系统，而是一套本地优先、面向真实工作的 AI 工作底座。

在 `v1.1.0` 中，我们把重点放在两件事上：

- 把销售和客服两条高频工作流做稳
- 把可复用的知识资产真正沉淀下来

这一版新增了少量高价值数字员工角色，并把它们严格绑定在具体流程阶段里，比如销售资格判断、跟进草稿、客服回复、风险复核和知识资产整理。这样做的目的不是“看起来更聪明”，而是为了更稳定、更准确、更容易人工审核。

同时，`Knowledge Vault` 不再只是一个静态存储页。它现在已经可以承接销售资产和 FAQ / 升级边界资产，并支持搜索、归档、编辑、一键复用和来源追溯。

如果你真正关心的是让 AI 参与日常业务，而不是只在对话框里演示一下，那么 `v1.1.0` 会比之前更接近一个能持续积累的工作系统。

## 4. 核心卖点

1. **销售和客服链路更稳**
   当前最成熟的是销售跟进、客服处理和知识资产沉淀。

2. **数字员工角色更克制**
   只保留少量高价值、可验证、可审计的专家角色。

3. **Reality Checker 加入关键复核节点**
   输出不是直接放行，而是先做风险检查。

4. **Knowledge Vault 进入可复用阶段**
   资产不再只是存起来，而是可以编辑、追溯、一键复用。

5. **本地优先**
   更适合重视可控性、隐私性和长期工作资产沉淀的用户。

6. **桌面壳 + 浏览器壳双入口**
   贴近真实工作方式，而不是只停留在网页试玩。
