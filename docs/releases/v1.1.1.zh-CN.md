# AgentCore OS v1.1.1 版本说明

`v1.1.1` 是当前稳定线上的一次维护版发布。

这一版不扩大产品边界，也不新增一套新的流程面。
它的目标很直接: 把现有 `v1.1.x` 这条稳定线修得更准、讲得更清楚、发得更稳。

## 版本定位

相对于 `v1.1.0`，这一版主要处理三件事：

1. 修正会导致流程资产复用计数偏早累加的问题
2. 让公开发布文档与真实兼容边界保持一致
3. 统一 GitHub 和 CNB 的对外发布入口与中文说明

这让 `v1.1.1` 更适合：

- 正在评估销售和客服闭环的团队
- 需要更清晰安装入口和发布说明的中文用户
- 希望先用稳定维护版，而不是继续叠加新功能的本地优先使用者

## 本版修复内容

### 1. Knowledge Vault 复用计数修正

`Knowledge Vault` 的流程资产一键复用，不会再在用户真正确认复用之前提前累计复用次数。

现在复用计数会和真实流程确认动作保持一致，减少统计偏差。

### 2. 公开发布边界表述修正

公开文档现在会更准确地描述 `openclaw` 到 `runtime` 的整理状态：

- 这是部分迁移
- 当前仍保留兼容层
- 不再把这件事写成“已经全部完成”

### 3. GitHub / CNB 双入口对齐

中文发布文档和对外说明已统一加入两套入口：

- GitHub 主仓库
- CNB 国内镜像

这样国内用户在查看发布说明时，不需要再自己猜测克隆入口。

## 保持不变的产品边界

`v1.1.1` 仍然坚持当前稳定线的核心判断：

- 本地优先
- BYOK / API Key 驱动
- 关键输出保持人工审核边界
- 销售、客服、知识资产沉淀仍然是当前最成熟的能力闭环

## 安装方式

当前推荐安装方式：**命令行安装 / 从源码运行**

```bash
git clone https://github.com/aidi1723/agentcore-os.git
cd agentcore-os
npm install
npm run dev
```

国内镜像：

```bash
git clone https://cnb.cool/aidiyangyu/agentcore-os
cd agentcore-os
npm install
npm run dev
```

推荐环境：

- Node.js `20+`
- npm
