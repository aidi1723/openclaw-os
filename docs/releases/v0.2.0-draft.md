# AgentCore OS v0.2.0 (Draft)

This is the current release draft for the next minor release after `v0.2.0-alpha.1`.

## Themes

- Business-solution-first entry instead of isolated tools
- Packaged scenario apps for real-world automation use cases
- Language-first onboarding for global users
- Desktop workflow maturity and local-first operation

## Release candidate highlights

### Product direction

- AgentCore OS is now positioned as a business solution operating system, not only a browser desktop shell.
- Users can enter by industry, switch scenarios, and load a matching workspace with pre-packaged apps.
- Mature use cases are mapped into app bundles through Industry App Center and Solutions Hub.

### Major additions in `main`

- Industry App Center with:
  - industry bundles
  - workspace builder
  - packaged desktop and dock presets
- Solutions Hub expanded around mature use-case mapping
- Multi-language onboarding:
  - top-level language switcher
  - first-launch language selection
- New packaged apps:
  - Recruiting Desk
  - Project Ops Board
  - Deep Research Hub
  - Financial Document Bot
  - Social Media Auto-pilot
  - Website SEO Studio
  - Morning Brief
  - Meeting Copilot
  - Personal CRM
  - Inbox Declutter
  - Support Copilot
  - Second Brain
  - Email Assistant
  - Deal Desk
  - Family Calendar
  - Habit Tracker
  - Health Tracker
  - Creator Radar
  - Content Repurposer
  - Tech News Digest
  - Language Learning Desk

### Existing platform upgrades carried forward

- Desktop UX: resizable windows + keyboard tiling / restore shortcuts
- Spotlight: recent history + improved command UX
- Playbooks: local-first save/export/import
- Publisher: queue, retry, dry-run, connector-based publishing

## Compatibility

No breaking changes planned (subject to change during implementation).

## Known limits

- This is still a pre-release branch state, not a hardened production release.
- Some OpenClaw-dependent health checks may depend on local gateway/runtime availability.
- Secrets and settings are still local-first demo behavior unless replaced in deployment.
