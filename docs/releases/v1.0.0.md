# AgentCore OS v1.0.0

`v1.0.0` is the first stable public release line for AgentCore OS.

This release marks the point where the project can be presented as a stable, local-first AI work platform for real workflows rather than a beta evaluation build.

## Release positioning

Compared with the earlier beta line, `v1.0.0` should now be treated as the main public release entry for the current open-source AgentCore OS distribution.

It is suitable for:

- public GitHub release distribution
- stable product evaluation
- desktop installer packaging and trial
- workflow validation in real work scenarios
- documentation indexing and onboarding

## What this release delivers

### 1. Stable product boundary

This release establishes a clearer stable product boundary across the browser shell, desktop shell, runtime-facing API surface, and execution-layer naming.

Public-facing positioning now centers on:

- local-first AI work execution
- BYOK / API Key driven setup
- desktop and browser dual entry
- real workflow usage instead of chat-only evaluation
- three stable core lanes: content creation, sales follow-up, and research analysis

### 2. Unified runtime and execution naming

This release continues the naming cleanup and execution-layer convergence work:

- frontend assistant naming is aligned with the runtime boundary
- LobsterAI remains the capability substrate, while the exposed execution identity is unified around the AgentCore OS runtime surface
- default execution paths, runtime APIs, desktop shell behavior, and selected bridge paths now follow the same public naming direction

### 3. Stable release gate verification

The stable release gate for the default `desktop_light` line was re-verified before publication. The validated path includes:

- `npm run build`
- `npm run runtime:doctor`
- `npm run test:stability`
- `npm run test:baseline`
- `npm run test:fresh-api-only`
- `npm run test:paperclip-adapter`
- `npm run desktop:build-doctor`
- `python3 -m py_compile lobster-sidecar/main.py lobster-sidecar/lobster_bridge.py`
- `npm run desktop:smoke-test-sidecar`

### 4. Platform baseline clarification

During release verification, the production build path was confirmed against Node.js 22 LTS. This is the recommended local development and release-validation baseline for the current stable line.
