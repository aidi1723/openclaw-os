# AgentCore OS v1.1.0

`v1.1.0` is the next stable release line for AgentCore OS after the first public stable baseline.

This version is centered on one product judgment:

AgentCore OS should prove value through stable, reviewable, reusable workflow execution, not by exposing a large number of loosely connected agent roles.

## Release positioning

Compared with `v1.0.0`, this release moves the product forward in three practical ways:

1. it makes the sales and support chains deeper
2. it introduces a small curated expert-role layer instead of a role explosion
3. it turns Knowledge Vault into a reusable process-asset layer rather than a passive storage view

This makes `v1.1.0` more suitable for:

- high-frequency sales follow-up work
- high-frequency support handling and FAQ accumulation
- teams that need human-review boundaries and reusable workflow assets
- local-first operators who care about stable accumulation more than flashy autonomy demos

## What this release delivers

### 1. Curated digital employee layer

This release adds a strict expert whitelist for stage-bound workflow help:

- `sales_qualification_specialist`
- `outreach_draft_specialist`
- `support_reply_specialist`
- `reality_checker`
- `knowledge_asset_editor`

These roles are not exposed as free-floating apps.
They are attached to concrete workflow stages where output quality can be reviewed and verified.

### 2. Stronger sales and support chains

The most mature workflow depth in this release now sits in:

- `Deal Desk`
- `Email Assistant`
- `Personal CRM`
- `Support Copilot`
- `Knowledge Vault`

Sales flow:

- inquiry intake
- qualification
- outreach draft
- human review
- CRM closeout
- reusable sales asset write-back

Support flow:

- issue intake
- reply draft
- reality check
- human review
- FAQ / escalation-boundary asset write-back

### 3. Knowledge Vault as a process-asset layer

`Knowledge Vault` now supports structured process assets with:

- search
- status filter
- archive / restore
- remove
- source jump
- reuse count
- one-click reuse
- in-place editing for title, scene, tags, and body
- visible asset lineage and timestamps

This is a key step because the system can now:

- produce assets
- store assets
- review assets
- edit assets
- reuse assets in the next run

### 4. Human-review boundary stays explicit

This release keeps the product conservative where it matters:

- expert roles are whitelist-based
- `Reality Checker` runs as a review-stage constraint
- outputs are still expected to be human-reviewed before external use
- assetization happens at workflow closure points, not as uncontrolled background generation

## Verification performed for this release

The release path verified in the current workspace for `v1.1.0` is:

- `npm run lint`
- `npm run build`

No broader end-to-end UI automation is claimed in this release note.
