# AgentCore OS v0.2.0-beta.2

`v0.2.0-beta.2` is the current public distribution release for AgentCore OS.

This version is the recommended starting point for external evaluation, desktop installation, workflow validation, and product understanding. It consolidates the public naming transition, desktop-runtime iteration, workflow-oriented product framing, and supporting documentation into one release baseline.

## Release positioning

Compared with earlier beta tags, `v0.2.0-beta.2` should now be read as the main public release entry for the current AgentCore OS open-source line.

It is suitable for:

- public GitHub release distribution
- external product evaluation
- desktop installer download and trial
- workflow and scenario validation
- documentation indexing and onboarding

## What this release delivers

### 1. AgentCore OS public product framing

This release stabilizes the public-facing AgentCore OS framing as a local-first AI work platform rather than a single-purpose AI tool.

The project direction is now clearly centered on:

- local-first AI execution
- privacy-conscious real-work usage
- desktop-based workflow orchestration
- multi-scenario adoption across sales, operations, support, content, research, programming, factory, warehouse, and data work
- accumulation of reusable workflow assets instead of one-off chat output

### 2. Desktop and workspace iteration

This release includes continued desktop-runtime and workspace improvements, including:

- clearer work-category-based desktop organization
- category-driven metadata across desktop and Settings
- better desktop overview and Dock counts
- richer workspace presets for multiple scenarios
- improved scenario-driven initialization paths
- continued Industry App Center and Solutions Hub alignment

### 3. Session continuity and usability fixes

The runtime experience is more stable and practical in daily use, including:

- session history restoration from local storage
- recovery of orphaned message history into visible sessions
- expanded retention for more sessions and preserved messages
- longer session-strip history navigation
- fixes for session isolation issues during conversation switching

### 4. Documentation and packaging readiness

This release also consolidates documentation and distribution readiness:

- `README.md` now points to the current recommended public version
- public release positioning is aligned in `docs/PUBLIC_RELEASE.md`
- release notes are suitable for direct GitHub Release usage
- Chinese documentation entry, early-access distribution notes, command-line install notes, and macOS install references are now part of the release surface
- desktop packaging outputs remain part of the distribution path for macOS and Windows
